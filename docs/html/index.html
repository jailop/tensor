Tensor4D Library Documentation

Introduction

The tensor4d library is a high-performance C++ tensor computation library with support for automatic differentiation, GPU acceleration, and BLAS integration. It provides a NumPy-like API for C++ developers.

Key Features

  • Multi-dimensional Tensors: Support for tensors of arbitrary rank
  • Automatic Differentiation: Built-in autograd for gradient computation
  • GPU Acceleration: CUDA support for tensor operations
  • BLAS Integration: Optimized matrix operations via BLAS
  • NumPy-like API: Familiar interface for Python developers
  • Type Safety: Compile-time shape checking for fixed-rank tensors
  • Pre-compiled Types: Common tensor types available as libraries

Getting Started

Installation

Build the library using CMake:

mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make
make install

Basic Usage

#include "tensor.h"
#include "tensor_types.h"
using namespace tensor4d;
int main() {
// Create a 3x3 matrix
Matrixf A({3, 3});
A.fill(1.0f);
// Create a vector
Vectorf v({3});
v.fill(2.0f);
// Matrix-vector multiplication
auto result = A.matmul(v);
// Access elements
std::cout << result[{0}] << std::endl;
}

Backends

The library automatically selects the best available backend:

  1. GPU (CUDA) - If CUDA is available
  2. BLAS - If BLAS/OpenBLAS is available
  3. CPU - Fallback implementation

Library Files

Two library files are built:

  • libtensor4d_static.a - Static library
  • libtensor4d.so - Shared library

Both contain pre-compiled instantiations of common tensor types. See Template Instantiations for details.
